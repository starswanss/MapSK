# ยลดาพาทัวร์ 🗺️ (Yolada Pa Tour)

แอปพลิเคชัน **แผนที่ + เกมท่องเที่ยว** เส้นทางจีโอพาร์คโคราช
นำเที่ยวโดยมาสคอต **ยลดา** และทีมผู้ติดตามอีก 3 ตัวละคร

## ✨ ฟีเจอร์
- **แผนที่แบบกดได้** (Leaflet + OpenStreetMap — ไม่ต้องใช้ API key) พร้อมหมุด 12 จุด และเส้นทางจีโอพาร์คที่เริ่มจากอนุสาวรีย์ท้าวสุรนารี
- **ตัวละครการ์ตูนอนิเมชัน 4 ตัว**
  - 🩷 **ยลดา** — ไกด์นำทาง/มาสคอต (เด่นที่สุด)
  - 💜 **เมษยา** — ประวัติศาสตร์และความเป็นมา
  - 🧡 **ปัญญา** — ธรณีวิทยาและฟอสซิล (เส้นทางจีโอพาร์ค)
  - 💚 **ธัญชนก** — Tips การเตรียมตัว/ช่วงเวลาที่เหมาะสม
- **แผงข้อมูลแต่ละสถานที่** กดหมุด/การ์ด แล้วสไลด์เปิด พร้อมแท็บข้อมูลจากตัวละครแต่ละตัว
- **เกมเก็บดวงตรา “พาสปอร์ตจีโอพาร์ค”** เช็คอินครบ 12 จุด → คอนเฟตตีฉลอง (บันทึกความคืบหน้าด้วย localStorage)
- **ระบบจัดการรูปในตัว** อัปโหลดรูปแต่ละจุดเองได้ เซฟเป็นไฟล์จริงลงโฟลเดอร์ `images/` (ไม่ต้องรันเซิร์ฟเวอร์หลังบ้าน/ไม่มีค่าใช้จ่าย)
- **5 อำเภอบนแผนที่** เลือกอำเภอ → ไฮไลท์โซน + แสดงสิ่งน่าสนใจ (อาหาร/สถานที่/วัฒนธรรม) และ **บทประพันธ์** ของแต่ละอำเภอ
- **เครื่องเล่นเสียงบทประพันธ์** แต่ละบทมีปุ่มเล่นเสียง + อัปโหลดไฟล์เสียงเองได้ (อยู่ในแท็บ “🎵 เสียงบทประพันธ์” ของหน้าจัดการรูป)
- **🧠 เกมทดสอบความรู้** 12 ข้อ (สุ่มหัวข้อละ 1 ใน 3) + เก็บคะแนน + ระบบจัดอันดับ (ใส่แค่ชื่อ คะแนนเท่ากันตัดสินที่เวลา)
- **⚔️ โหมดผจญภัย** เล่าเรื่องโดย “แม่หน่อย” 12 ด่าน แต่ละด่านมีมินิเกมไม่ซ้ำกัน เก็บกุญแจพลังครบ 12 ดอก → รับเหรียญ “Korat Geopark Guardian”
- ปุ่ม **นำทาง Google Maps** จริงในแต่ละสถานที่
- รองรับมือถือ/แท็บเล็ต และ `prefers-reduced-motion`

### ใช้รูปตัวละครจริงของตัวเอง
ตอนนี้ตัวละครเป็น **SVG การ์ตูนน่ารัก** ที่วาดด้วยโค้ด (ฟังก์ชัน `mascotSVG()` ใน `app.js`)
หากต้องการใช้รูปจริง ให้แก้ `mascotSVG()` ให้คืนค่าเป็น เช่น
`<img src="characters/yolada.png" width="..." />` แทน

### รูปสถานที่จริง

แอปเลือกแสดงรูปตามลำดับ: **รูปที่อัปโหลด/ใส่เอง → รูปจาก Google Places (ถ้าตั้งคีย์) → ภาพประกอบ SVG**
ปุ่ม **“จัดการรูป”** มุมขวาบนทำงานได้ 2 โหมด เลือกอัตโนมัติจาก `config.js`

#### ⭐ โหมดออนไลน์ (Supabase) — แนะนำเมื่อขึ้น Netlify / มีหลายคนใช้
อัปโหลดบนเว็บจริงได้เลย ทุกคนเห็นรูปเดียวกัน ฟรี ไม่ต้องใช้บัตรเครดิต ไม่ต้องรันเซิร์ฟเวอร์เอง

**ตั้งค่า Supabase (ครั้งเดียว ~5 นาที):**
1. เข้า [supabase.com](https://supabase.com) → **New project**
2. **Storage → New bucket** → ชื่อ `places` → ติ๊ก **Public bucket** → Create
3. **Project Settings → API** → คัดลอก **Project URL** และ **anon public key** ไปวางใน `config.js` (`SUPABASE_URL`, `SUPABASE_ANON_KEY`)
4. **Storage → Policies** → ที่ bucket `places` เพิ่ม policy ให้ role `anon` ทำได้ (กดหน้า SQL Editor วางคำสั่งนี้):

```sql
-- อนุญาตให้ทุกคนดู/อัปโหลด/แก้/ลบไฟล์ใน bucket "places"
create policy "yolada read"   on storage.objects for select using ( bucket_id = 'places' );
create policy "yolada insert" on storage.objects for insert with check ( bucket_id = 'places' );
create policy "yolada update" on storage.objects for update using ( bucket_id = 'places' );
create policy "yolada delete" on storage.objects for delete using ( bucket_id = 'places' );
```
5. (แนะนำ) ตั้ง `ADMIN_PASSCODE` ใน `config.js` เป็นรหัสง่าย ๆ กันคนสุ่มแก้รูป

> 🔓 **เรื่องความปลอดภัย:** policy ด้านบนเปิดให้ใครก็อัปโหลด/ลบได้ (เหมาะกับงานกลุ่ม/ทดลอง)
> ถ้าต้องการคุมเข้มขึ้นจริงจัง ให้ใช้ Supabase Auth แล้วเปลี่ยน policy ให้เช็ก `auth.role() = 'authenticated'`

เสร็จแล้วกด **“จัดการรูป” → “อัปโหลดรูป”** ได้เลย ไม่ต้อง push โค้ดใหม่ รูปจะขึ้นให้ทุกคนทันที

#### โหมดในเครื่อง (ไม่ตั้ง Supabase) — เซฟไฟล์ลงโฟลเดอร์ images/
ใช้ตอนรันในเครื่องด้วย **Chrome/Edge**: กด “จัดการรูป” → “เชื่อมต่อโฟลเดอร์ images” → อัปโหลด
ไฟล์จะถูกเซฟลง `images/` จริง (`place-01.jpg` …) + `images/manifest.json` แล้วค่อย `git push` ขึ้น repo

**(ทางเลือก) ใส่ไฟล์รูปเองใน data.js**
วางไฟล์ในโฟลเดอร์ `images/` แล้วเพิ่มฟิลด์ `image` ในสถานที่นั้นใน `data.js`:

```js
{
  id: 9,
  name: "เขื่อนลำตะคอง",
  image: "images/lamtakhong.jpg",   // ← เพิ่มบรรทัดนี้ ระบบจะใช้รูปจริงแทนภาพประกอบทันที
  ...
}
```
ระบบจะแสดงรูปจริงทั้งในแผงรายละเอียดและในรายการสถานที่โดยอัตโนมัติ

**(ตัวเลือกเสริม) ดึงรูปอัตโนมัติจาก Google Places**
ต้องมี API key + เปิด billing — ดูขั้นตอนในไฟล์ `config.js` (ถ้าไม่ตั้งคีย์ ส่วนนี้จะไม่ทำงาน ไม่กระทบอะไร)

## 🚀 ขึ้น GitHub + Netlify
1. ตั้งค่า **Supabase** ใน `config.js` ให้เรียบร้อยก่อน (ดูหัวข้อด้านบน) — เพื่อให้การอัปโหลดรูปทำงานบนเว็บจริงสำหรับทุกคน
2. push โค้ดทั้งโฟลเดอร์ขึ้น GitHub
3. เข้า [netlify.com](https://netlify.com) → **Add new site → Import an existing project** → เลือก repo
   - Build command: *(เว้นว่าง)* · Publish directory: `.` (มี `netlify.toml` ตั้งให้แล้ว)
4. Deploy เสร็จ → เปิดเว็บ → กด “จัดการรูป” อัปโหลดรูปได้เลย ทุกคนที่เข้าเว็บเห็นรูปเดียวกัน

> ⚠️ ถ้า **ไม่ได้ตั้ง Supabase** แล้ว deploy ขึ้น Netlify การกดอัปโหลดจะเขียนลงเครื่องคนกดเท่านั้น (ไม่อัปเดตเว็บจริง) เพราะ Netlify เป็น static hosting แบบอ่านอย่างเดียว

### เปลี่ยนเป็น Google Maps จริง (ต้องมี API key)
ตอนนี้ใช้ Leaflet + OpenStreetMap เพราะ **ไม่ต้องใช้ API key** และฟรี
ปุ่ม “นำทาง” ลิงก์ไป Google Maps จริงอยู่แล้ว หากต้องการฝังแผนที่ Google Maps แทน Leaflet
สามารถใส่ Google Maps JavaScript API แล้วแทนที่ส่วน `initMap()` ใน `app.js`

> หมายเหตุ: พิกัดสถานที่เป็นค่า **โดยประมาณ** เพื่อการสาธิต ปรับให้ตรงจุดจริงได้ใน `data.js`
